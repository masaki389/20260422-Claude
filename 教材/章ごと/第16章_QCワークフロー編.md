# 第16章：QCワークフロー編
### AIが作った画像を仕分ける仕組みと、問題を「台本設計」で事前につぶす技術

---

AIで画像を作るのは数秒です。でも「使えるかどうか」を判断して、ダメな画像を直して、次回の生成に活かすまでのループがなければ、品質は永遠に上がらない。

この章では、実際の幸子チャンネル第6話「老後2000万円問題」の制作で作り上げたQCワークフローを完全公開します。

---

## なぜQCが必要か

AI画像生成は「確率」の世界です。

同じプロンプトを10回送っても、返ってくる絵は10通りです。背景が変わる。顔が変わる。服が変わる。これは仕様です。

だから「プロンプトさえ完璧にすればいい」という発想は半分しか正しくない。

完璧なプロンプトを書いても、30%は微妙な絵が出る。
80%を60%に上げるためにプロンプトを磨く努力より、
出てきた80枚の中から使える60枚を選ぶ目を育てるほうが、ずっとコスパがいい。

「生成で解決しようとする」から「設計と選別で解決する」へ。これがQCワークフローの本質です。

---

## ステップ1：スプシでNG管理する

第6話制作では、120枚の画像をGoogleスプシでNG管理するシステムを作りました。

### スプシの列構成

| 列 | 内容 |
|---|---|
| A | カット番号（cut_001） |
| B | 台本の場所・内容 |
| C | 生成プロンプト |
| D | 画像URL（Drive直リンク） |
| E | NG / 要修正 / OK |
| F | NGの理由（自由記述） |
| G〜J | 改善前後の比較用（v2プロンプトを追加） |
| K | 監督メモ（自由記述） |

### NGの理由に使う分類

判定を速くするために、NGの理由を分類しておくと後でプロンプト改善に使いやすい。

- **背景NG**：別の部屋になっている・家具の色が変わっている
- **服装NG**：幸子が職場服で家にいる・花柄ブラウスのはずが無地になっている
- **別人化**：顔の造形が変わっている・髪型が変わっている
- **構図NG**：引きの指示なのにアップになっている（または逆）
- **内容NG**：台本の内容が全く反映されていない

### スプシをClaudeに読ませる

NGを記入したスプシのURLを貼って「スプシ確認して」と言えば、Claudeが全件読んでプロンプト改善案を出します。

実際に第6話のカット1〜17を確認したときに出た分析結果の例：

> - カット3・7・11：背景NGが集中 → カフェシーンで参照画像なし。マスターショット追加が必要
> - カット5・9：服装NG → `is_home_scene`バグが原因。職場制服が自宅シーンに適用されていた（修正済み）
> - カット14・15：別人化 → 久美のキャラ参照画像が未登録のためモデルが自由生成

「バグ」と「プロンプトの問題」と「そもそもの設計問題」を分けて診断できるのが最大の利点です。

## ステップ2：プロンプトより先に「台本設計」で直す

スプシのNGを見ていると、あることに気がつきます。

**同じシーンで背景が変わる問題の9割は、プロンプトでは解決できない。**

Geminiは各カットを独立して生成します。前のカットの画像を「記憶」していません。どれだけ「前の背景と同じにして」と書いても、モデルには前の絵が見えていない。

では何をすべきか。

**答えは「そのカットの生成依頼に、前の絵を添付する」か「そもそも同じ絵を使い回す台本を書く」のどちらかです。**

### 再利用設計の考え方

幸子チャンネルでカフェ会話シーンが出てきたとき、最初はすべてのカットを新規生成しようとしていました。結果、カット毎に背景がバラバラになった。

正しいアプローチはこうです。

**カフェ会話シーンで必要な「基準画像」は3枚だけ。**

```
1. 2ショット（引き）：幸子と久美が向かい合って座っている全体像
2. 幸子アップ：幸子の顔アップ
3. 久美アップ：久美の顔アップ
```

この3枚を「命がけで整合性を持たせて」生成します。

残りの同じシーン内のカットは、この3枚の使い回しでOKです。

なぜ使い回しでいいのか。CapCutのAI動画化（Seedance）を使うと、静止画でも10〜15%の動きがつきます。同じ画像でも、毎回微妙に違う動きになるので、視聴者には「カットが変わっている」ように見えます。

使い回すカットは台本に `再利用：outputs/cut_XXX.png` と書くだけです。

```
【カット8】
場所：カフェ
キャラ：幸子（アップ）
内容：幸子が笑ってコーヒーを飲む
カメラ：バストアップ
秒数：3
再利用：outputs/cut_006.png
```

生成コスト0円。背景一致率100%。

## ステップ3：`シーン参照：` フィールドで継続性を保つ

新規生成が必要なPOVカット（逆アングル・ドリーイン等）には、`シーン参照：` を使います。

```
【カット9】
場所：カフェ
キャラ：久美（アップ）
内容：久美が少し得意げな顔で微笑む
カメラ：バストアップ
秒数：3
シーン参照：outputs/cut_004.png
```

`cut_004.png` がカフェ2ショットのマスターショットだとすると、カット9を生成するときにそのマスターショットが「このシーンの空間参照」としてGeminiに同時送信されます。

Geminiへの指示はこうなります：

> SCENE CONTINUITY REFERENCE (CRITICAL — TOP PRIORITY): This is the master shot for this conversation scene. You MUST match the following elements with 100% fidelity: table shape, table surface color and texture, chair design and color, window position and size, wall color, lighting direction and warmth, and the relative seating positions of all characters.

これによって、POVカットでも「同じカフェ」「同じ席」であることを担保できます。

### 期待改善効果の目安

| 問題 | 改善前 | シーン参照あり |
|---|---|---|
| 背景の一致率 | 50〜60% | 80〜90% |
| POV整合（幸子→久美の逆アングル） | 0〜10% | 60〜70% |
| 照明の色統一 | 40〜50% | 70〜80% |

「完全一致」は難しい。でもそれは使い回しでカバーする。新規生成が必要な場面だけシーン参照を使う、という役割分担が最適解です。

## 3点セット法：会話シーンの最小単位

会話シーンの品質戦略をまとめると、「3枚に命をかける」という原則になります。

**3点セットの構成：**

```
1. マスターショット（引き・2ショット）
   → 空間情報のすべてを含む基準画像

2. キャラAのアップ（正面 or バストアップ）
   → シーン参照 = マスターショット

3. キャラBのアップ（正面 or バストアップ）
   → シーン参照 = マスターショット
```

この3枚が揃えば、同じシーン内の他のカットはすべて「再利用」か「シーン参照あり生成」で賄えます。

**script.txtの書き方例（カフェ久美シーン）：**

```
【カット4】
場所：カフェ
キャラ：幸子　久美
内容：2人が向かい合ってコーヒーを飲みながら話している
カメラ：ミディアム
秒数：4

【カット5】
場所：カフェ
キャラ：幸子（アップ）
内容：幸子が苦笑いをする
カメラ：バストアップ
秒数：3
シーン参照：outputs/cut_004.png

【カット6】
場所：カフェ
キャラ：久美（アップ）
内容：久美が「うちの主人、それの倍以上あるのよ」と言う
カメラ：バストアップ
秒数：3
シーン参照：outputs/cut_004.png

【カット7】
場所：カフェ
キャラ：幸子（アップ）
内容：幸子がうつむく
カメラ：バストアップ
秒数：3
再利用：outputs/cut_005.png
```

カット4が3点セットのマスター。カット5・6が新規生成（シーン参照あり）。カット7は使い回し。

合計3枚の新規生成で、4カットが成立します。

## ステップ4：フィードバックループを回す

QCワークフローの最終形は、こういうサイクルです。

```
台本設計（3点セット法 + 再利用設計）
     ↓
auto_studio.py で生成
     ↓
Drive にアップ
     ↓
監督がスプシに NG / OK / 要修正 + メモを記入
     ↓
「スプシ確認して」でClaude がNG分析
     ↓
プロンプト改善 or バグ修正 or 台本設計の見直し
     ↓
次話の制作に反映
     ↓（繰り返し）
```

このループを回すたびに、次の話の一発OKカット数が増えていきます。

第6話で出た問題（背景バラつき・服装ミス・別人化）は、すでにプロンプトと台本ルールに反映しています。第7話では同じ問題は起きません。

新しい問題が出る。それをスプシに記録する。ループが閉じる。

これが「AIアニメ制作の品質管理」の実体です。

---

## まとめ

AIアニメの品質管理で大事なことは3つだけです。

1. **出てきた絵を記録して仕分ける**（スプシQCシステム）
2. **プロンプトより台本設計で先に回避する**（3点セット法 + 再利用設計）
3. **どうしても新規生成が必要なカットだけ、シーン参照で継続性を保つ**（`シーン参照：` フィールド）

「全部プロンプトで解決しようとする」のをやめると、制作スピードが一気に上がります。

生成枚数を減らしながら、品質を上げる。その矛盾を解くのが「設計」です。

## 付録：script.txt に追加できるフィールド一覧

```
【カットN】
場所：自宅 / レジ / 休憩室 / カフェ など
キャラ：幸子（アップ）/ 正夫 / 久美 など
内容：シーンの説明（ナレーションに出てくる要素1つ）
カメラ：引き / ミディアム / バストアップ / クローズアップ / 横顔
秒数：4
再利用：outputs/cut_XXX.png    ← このカットを使い回す
プロップ：assets/prop/XXX.png  ← 小道具の参照画像
シーン参照：outputs/cut_XXX.png ← 空間継続性のためのマスターショット参照
```

`再利用` と `シーン参照` の使い分け：

- **再利用**：同じ絵でいい。コスト0。背景100%一致。
- **シーン参照**：角度や表情が変わるので新規生成が必要。でも同じ空間でないといけない。
