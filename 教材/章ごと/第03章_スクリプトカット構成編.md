# 第3章：スクリプト・カット構成編
### 10分動画に120カット必要な理由

---

AIアニメを始めた頃、カット数の概念がよく分かっていませんでした。

「5〜6枚の画像を繋げれば動画になる」くらいの感覚で作っていたんですよね。実際に最初に作ったとき、1つのシーンを1〜2枚で表現していた。

編集してみて気づいたのは、素材が全然足りないということでした。ナレーションが「残り物3つ、使いかけの調味料5本、干からびたネギ、期限切れ納豆」と続くのに、「冷蔵庫の中」という画像1枚しかない。それに1分以上乗せるとどうなるかというと、同じ画像がずっと表示され続ける。

視聴者は飽きます。当然です。

---

## 「名詞・アクション1つ = 1カット」の鉄則

これが分かってから制作が変わりました。

ナレーションの中に出てくる「名詞」と「アクション」を数えて、それと同じ数だけカットを用意する。これだけです。

例を見た方が分かりやすいと思うので、実際に失敗したシーンで説明します。

**NG（1カットに詰め込みすぎ）：**
```
【カット5】
場所：自宅
キャラ：幸子（アップ）
内容：冷蔵庫の中を確認。残り物3つ、調味料5本、干からびたネギ、期限切れ納豆
```

これだと1枚の画像で全部表現しようとしている。でも1枚の画像に「残り物・調味料・ネギ・納豆」を全部写すと、それぞれがどれなのか分からなくなるし、AIも混乱する。

**OK（名詞・アクションごとに分割）：**
```
【カット5】
場所：自宅
内容：冷蔵庫を全開にした幸子の視点。棚に残り物が散らばっている（引きのショット）
秒数：3

【カット6】
場所：自宅
内容：残り物3つのクローズアップ。ラップに包まれた白いご飯、みそ汁の鍋、残り野菜
秒数：3

【カット7】
場所：自宅
内容：使いかけの調味料5本のクローズアップ。醤油、みりん、ポン酢などが並んでいる
秒数：3

【カット8】
場所：自宅
内容：干からびたネギのクローズアップ。萎れて茶色くなっている
秒数：2

【カット9】
場所：自宅
内容：賞味期限切れ納豆のパック。1ヶ月前の日付が見える
秒数：2
```

1文が5カットになりました。でもこれが正しい。

---

## カット数の目安

これを踏まえると、カット数の目安はこうなります。

- 3分のナレーション → 30〜40カット
- 5分のナレーション → 50〜70カット
- 10分のナレーション → **100〜120カット**

10分動画を作ろうとしたとき、最初は「せいぜい40〜50枚くらいかな」と思っていたんですが、実際に名詞・アクション単位で分解すると余裕で100枚を超えます。

「多すぎない？」と思うかもしれないですが、120カット使って初めて「テンポが良い10分動画」になる。これが少ないと、ひとつの画像が10秒以上表示され続ける場面が出てきて、視聴者が離脱します。

---

## カット順の基本：引き→クローズアップ

カットの順番にも原則があります。

**必ず「引き（全体）→クローズアップ（個別）」の順で入れる。**

先ほどの冷蔵庫の例で言うと、まず「冷蔵庫全体の引きショット」から入って、その後に「残り物のクローズアップ」「ネギのクローズアップ」と続けました。

これを逆にするとどうなるかというと、視聴者が「今どこにいるのか」「何のクローズアップなのか」が分からない状態になります。

映画や普通のドラマでも、シーンが始まるときはまず場所を見せてから登場人物を見せる、という構成になっていますよね。それと同じ話です。

**シーン開始のパターン：**
1. 場所全体の引きショット（3〜4秒）
2. 人物または重要な物のミディアムショット（3秒）
3. 感情・細部のクローズアップ（2〜3秒）

この流れを崩さないだけで、映像のテンポが格段に良くなります。

## テキスト画面を多用する罠

もう一つ、転職チャンネルを作り始めた頃にやってしまったことがあります。

「年収558万円」「ボーナス年2回」「退職金2,000万円〜3,000万円」みたいな数字を、テキストだけが書かれたスライドのカットで表現していたんですよね。初期の台本では16カット中8カットがテキスト画面でした。

これの何が問題かというと、せっかくアニメで作っているのに、テキスト画面だと「動画ではなくスライド」になってしまう。視聴者はアニメを見にきているはずなのに、突然Googleスライドみたいな画面が出てくる。

テンポも悪くなるし、YouTubeのアルゴリズム的にも「低クオリティコンテンツ」と判定される可能性があります。

**解決策は、数字・情報をアニメシーンの中で表現すること。**

```
NG：テキスト画面「ボーナス年2回・合計約100万円」

OK：会社の事務所。上司からボーナス封筒を両手で受け取る田中悠斗。
　　満足そうな表情。封筒を両手で丁寧に受け取っている。
```

「100万円の封筒を受け取る」という場面を作れば、ナレーションが「ボーナスは年2回、合計約100万円」と言った瞬間に映像と言葉が合う。テキストスライドよりずっと自然に伝わります。

## 台本フォーマットと制作スクリプトの関係

カット割りを書くときは、後で画像生成スクリプトが読み込めるように、フォーマットを統一しておくと効率が上がります。

僕が使っているフォーマットはこれです。

```
【カット1】
場所：パン工場・外観・深夜
キャラ：なし
衣装：なし
内容：深夜も煌々と明かりが灯る白い金属パネルの現代的な大型工場建物。広いドックに赤と白の配送トラックが並んでいる。
カメラ：引き・広角・夜
秒数：3
```

項目は「場所・キャラ・衣装・内容・カメラ・秒数」の6つ。

**場所** はロケーションのキーワード。同じ場所名のカットには同じ背景を参照させるために使います。
**キャラ** に「（アップ）」を付けると顔アップ用の参照画像を使う設計にしています。
**衣装** はキャラがいるカットで制服を指定する欄。
**秒数** を書いておくと、ナレーションの尺と合わせやすい。

このフォーマットを崩すと、後で画像生成を自動化しようとしたときに読み込めなくなるので、一度決めたら統一することをおすすめします。

---

## 【補足】50〜60秒のShortsが最も完視聴率が高い

外部データで分かったことですが、YouTubeショートは**50〜60秒の長さが完視聴率76%**で最も高いという結果が出ています。

転職チャンネルの設計を見ると、16カット × 3〜4秒 = 48〜64秒というのがちょうどこの範囲に入っています。

これが意図せず正解のレンジに収まっていたわけで、カット数を増やしすぎて70秒・80秒になると完視聴率が下がる可能性があります。**転職ショートの目標長は50〜60秒**を意識して設計するのが良さそうです。

---

## まとめ

カット構成のポイントをまとめます。

1. **名詞・アクション1つ = 1カット**
   1文に複数の名詞やアクションが出てきたら分割する

2. **10分動画は100〜120カット**
   「多すぎる」は存在しない。少なすぎる方が問題。

3. **引き→クローズアップの順番を守る**
   場所を見せてから人物・細部を見せる

4. **テキスト画面は使わない**
   数字・情報はアニメシーンの中で表現する

5. **台本フォーマットを統一する**
   後で自動化するときに崩れてしまう

6. **転職ショートは50〜60秒が完視聴率のスイートスポット**
   16カット × 3〜4秒で自然にこの範囲に入る
