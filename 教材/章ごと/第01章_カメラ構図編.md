# 第1章：カメラ・構図編
### AIアニメを作ってて最初にぶつかる壁

---

AIアニメを始めた頃、僕は正直なんとなくで作っていました。画像を生成して、CapCutで繋いで、投稿する。それだけでした。

でも実際に作り続けていくと、AIには聞いても絶対に返ってこない知識が積み上がっていくんですよね。

この章では、その中でも最初に必ずぶつかる「カメラ・構図」の落とし穴について話します。カメラワークって聞くと難しそうですが、要するに「どの角度から・どんな距離で撮るか」の指示の話です。これを間違えると、何十枚も作り直すことになります。実際に僕がなりました。

---

## 落とし穴①：運転席カットで道路が横向きになる

### 何が起きたか

冷凍トラックドライバーの転職紹介動画を作っていたときのことです。

「田中悠斗がトラックを運転しながら語りかける」シーンを作りたくて、プロンプトを書いたんですよね。「田中悠斗が運転席でハンドルを握り、カメラ目線で語りかけている。背後に高速道路が続く」みたいな感じで。

生成してみたら、**道路が横向きに走っていました。**

画面に対して垂直に、つまり左から右に道路が伸びているんです。普通、運転席から前を見たら道路は奥へと続いていくはずなのに、それが横向きになる。

最初は「プロンプトの書き方が悪かったかな」と思って直しました。「道路は奥へ続いている」「一点透視法で」「遠近感を出して」と細かく指示を加えて再生成。

**同じでした。**

何回やっても横向きの道路が出てくる。「なんで？」と思いながら10回くらいリトライしたんですが、結局全部同じ結果でした。

---

### なぜ起きるのか

これ、プロンプトの問題じゃないんですよね。

Geminiが苦手とする空間認識の問題です。「正面を向いたキャラクター＋背後に奥へ伸びる道路」という3次元的な関係を正しく描くことが、現状のモデルでは難しい。

言い換えると、**キャラクターを正面に置こうとすると、背後の空間がおかしくなる**んです。正面向きのキャラクターに引っ張られて、背後の道路も「正面向き」になってしまう。

これ、何百回プロンプトを直しても解決しません。モデルの構造的な問題なので。

「もっと上手い表現があれば解決できるはず」と思って粘ると、時間とお金を無駄にします。僕はそれをやってしまいました。

---

### 解決策：「語りかけは車の外」に出す

解決策はシンプルで、**語りかけシーンは車の外に出すこと**です。

```
NG：田中悠斗が運転席でハンドルを握り、カメラ目線で語りかけている。背後に高速道路が続く。

OK：深夜のサービスエリア駐車場。大型冷凍トラックを背にして立つ田中悠斗。
　　カメラ目線で語りかける。背後に白い大型トラックと夜の駐車場。
```

これで解決します。

「でも実際に運転しながら語りかけてるシーンが欲しい」という場合は、語りかけシーンと運転シーンを別のカットに分けることで対応できます。語りかけは車の外、運転の臨場感は別カットで出す。

もう一つ、語りかけシーンで使えるパターンとして「サービスエリアでトラックの前に立ち、指差しながら語りかける」もあります。これも自然に見えるので、CTAカット（最後の締めのシーン）でよく使っています。

---

### 運転中のシーンはどうするか

語りかけは外に出せばいいとして、「ハンドルを握って運転している様子」も当然必要になります。

これにも正解があって、**横顔・斜め後ろからのショット**にすること。

```
NG：田中悠斗が正面を向いてハンドルを握っている。背後に道路が続く。

OK：田中悠斗の横顔。前方を力強く見つめながらハンドルを握っている。
　　サイドウィンドウ越しに深夜の高速道路が見える。助手席に同僚ドライバーが座っている。
```

横顔にすることで、道路は「サイドウィンドウ越し」に見える形になります。これだとAIが空間関係を正しく処理できる。

## 落とし穴②：窓越し構図で体が乗り出す

### 何が起きたか

これは運転席問題を「なんとか修正できないか」と試行錯誤していたときに踏んだ失敗です。

「車内から窓越しに語りかければいいんじゃないか」と思って、「田中悠斗が運転席の窓から体を出してカメラに話しかけている」みたいなプロンプトを試しました。

出てきたのは**窓から体を乗り出して後ろを振り向いているキャラクター**でした。

見た目がかなり危険です。高速道路を走るトラックの窓から体を乗り出して後ろを見ている人間、という絵になる。視聴者に見せられるクオリティじゃないですし、そもそも交通安全的にどうかという話にもなる。

---

### なぜ起きるのか

「窓越し」という指示が、AIに「窓の外に出る」と解釈されてしまうんですよね。

車の窓越しに語りかけるという動作を人間は自然に理解できますが、AIにとっては「窓 + 体を出す + 後ろを向く」という組み合わせで処理されてしまう。

これもプロンプトで修正できると思って色々試したんですが、どうしても変な角度になる。結局この構図は完全に諦めました。

---

### 解決策：窓越し構図は使わない

これの解決策は「使わない」一択です。

語りかけシーンで自然に見える構図は今のところ2パターンだと思っています。

**パターン1：車外に出て立っている**
```
駐車場でトラックを背にして立つ田中悠斗。
カメラ目線で語りかける。背後に白い大型トラック。
```

**パターン2：運転中は横顔（語りかけではなく雰囲気のカット）**
```
田中悠斗の横顔。夜明けの空を見ながら前方を見つめる。
サイドウィンドウ越しに朝焼けが見える。
```

語りかけカットで「乗り物の中にいる感じ」を出したいなら、前後に運転中カットを挟むことで演出できます。語りかける場所は外でも、前後の文脈で「運転中」と視聴者は認識してくれます。

---

## この2つの落とし穴、共通していること

改めて整理すると、これらの問題に共通しているのは「AIが3次元空間の関係性を処理できない場面がある」ということです。

特に「前を向いている人物＋背後に奥へ伸びる空間」の組み合わせが弱い。

これはトラックに限らず、乗り物全般に起きます。電車の運転席、バイク、船の操縦席。「正面向きキャラクター＋背後の奥行き」という構図は避けた方が無難です。

**運転系のカット設計の原則：**
- 語りかけ → 車外に出す
- 運転中 → 横顔・斜め後ろから
- 窓越し → 使わない

この3つを守るだけで、作り直しが大幅に減ります。

---

## よく使うカット種別と使い分け

ここで、一般的なカット種別も整理しておきます。台本を書くときに毎回使うので覚えておいて損はないです。

| カット名 | 被写体の範囲 | 使いどき |
|---|---|---|
| 引き（ワイドショット） | 場所全体・人物が小さく | シーンの最初、場所説明 |
| ミディアムショット | 腰から上 | 作業シーン、会話シーン |
| バストアップ | 胸から上 | 語りかけ、感情表現 |
| クローズアップ | 顔・手など | 感情の強調、重要なもの |

**鉄則：引き→クローズアップの順で入れる。**

場所が分からないままクローズアップが始まると視聴者が混乱します。「今どこにいるのか」を引きで見せてからクローズアップで感情や細部を見せる、という順番を守るだけで映像のテンポが格段に良くなります。

## 落とし穴③：手元クローズアップの連続で画風が崩れる

### 何が起きたか

幸子チャンネルの台本で、計算用紙や電卓のクローズアップを連続で並べたことがありました。「数字の重みを見せたい」という意図だったんですが、実際に生成してみると、**顔が映らないカットだけ画風が他と全然違う**という現象が起きました。

具体的には、他のカットは統一されたアニメ風の絵柄なのに、手元だけのクローズアップは妙に写実的な、別の作品から持ってきたような絵になっていたんです。しかも人物の年齢や雰囲気まで変わって見える。

### なぜ起きるのか

これも構造的な問題です。キャラクターの顔が映らないカットは、**顔の参照画像が実質効かない**状態になります。AIにとって「このキャラの顔はこういう特徴」という強い制約が働かないので、画風そのものが自由になり、結果的に他のカットと一致しない絵が出やすくなります。

つまり、**顔が映る・映らないかで生成の安定度がガラッと変わる**ということです。

### 解決策：アニメ業界の「インサートカット」の考え方を借りる

これ、実はアニメ業界の編集技法である「インサートカット（情景カット）」の知識で解決できることに気づきました。

インサートカットとは、シーンの途中に手元や情景の別カットを挟む編集技法です。本来は「会話の繋ぎを自然にする」ための技法ですが、これをAIアニメに応用すると、

```
NG：計算用紙クローズアップ → 電卓クローズアップ → 計算用紙クローズアップ
　　（手元だけが3カット連続。どれも画風が崩れるリスクがある）

OK：計算用紙クローズアップ → 窓の外の夕景（情景カット） → 電卓クローズアップ
　　（情景カットを挟むことで、手元クローズアップの総数自体を減らせる）
```

情景カット（窓の外・時計・観葉植物など）は、そもそも人物が映らないカットなので「画風の崩れ」のリスク自体が違う種類になります（背景だけなので、キャラの一貫性問題は起きない）。**手元クローズアップを減らして情景カットに置き換えるだけで、不安定なカットの総数を減らせる**わけです。

### 上手・下手（かみて・しもて）も使える

ついでに、アニメ演出の基本である「上手・下手」の概念も台本設計に使えます。

舞台・映像の世界では、画面右（上手）にいるキャラは優位・上昇、画面左（下手）にいるキャラは劣位・転落を示唆すると言われています。セリフで説明しなくても、立ち位置だけで力関係を伝えられる。

幸子チャンネルなら、休憩室で田中さんと幸子が話すシーンで、最初は田中さんを上手（優位）に置き、終盤で立ち位置を入れ替える、といった演出ができます。プロンプトに「画面右に立つ」「画面左に立つ」と一言加えるだけなので、コストはゼロです。

## まとめ

この章で話したことをまとめます。

1. **運転席語りかけカット → 車外に出す**
   プロンプトでどう頑張っても道路が横向きになる。構図を変えるのが唯一の解決策。

2. **窓越し構図 → 使わない**
   危険な見た目になる。語りかけは外に出す。

3. **運転中カット → 横顔・斜め後ろから**
   サイドウィンドウ越しに景色を見せる構図が安定する。

4. **カット順は引き→クローズアップ**
   場所説明してから感情・細部。この順番を崩さない。

5. **手元クローズアップが連続するときは情景カットを挟む**
   顔が映らないカットは画風が崩れやすい。窓の外・時計などのインサートカットに置き換えて総数を減らす。

6. **上手・下手で力関係を演出する**
   画面右＝優位、画面左＝劣位。セリフなしで関係性の変化を見せられる。

これ、読んでしまえば「そういうことか」と思うかもしれないですが、知らないまま作り始めると必ず踏みます。僕は踏みました。同じ時間とお金を使わないための知識です。
