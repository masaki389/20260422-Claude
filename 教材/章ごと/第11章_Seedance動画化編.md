# 第11章：Seedance動画化編
### 画像を「動かす」プロンプトの書き方

---

Seedanceは画像に動きをつけて動画にするツールです。CapCutに内蔵されていて、今のところアニメ動画制作の動画化ステップとしては一択に近い。

ただ、プロンプトの書き方を知らないと「ほとんど動かない静止画に近い動画」しか出てこなくて、動画化した意味が薄くなります。

この章では「どう書けば意図した動きが出るか」を話します。

---

## Seedanceのプロンプトは「映像の動き」を書く

一番大事な前提として、Seedanceに渡すプロンプトは「静止画の説明」ではなく「映像の動き」を書くものです。

```
NG（静止画の説明）：
「深夜の大型パン工場の外観。白い金属パネルの建物。赤と白のトラックが並んでいる。」

→ 動きの指示がないので、画像がほぼそのまま動画になる。
```

```
OK（映像の動きを指示）：
「カメラがゆっくりと建物に向かってプッシュイン。
照明がほんのりと揺らめく。夜風でトラックの脇に置かれた旗が少し揺れる。
静かだが生きている工場の雰囲気。」
```

どんな動きが起きるかを具体的に書くことで、意図した動画が出やすくなります。

---

## カメラムーブの指示語

カメラの動き方を指示する言葉を覚えておくと便利です。

| 動き | 英語 | 効果 |
|---|---|---|
| ゆっくり近づく | slow push in / dolly in | 臨場感・緊張感 |
| 引いていく | pull out / zoom out | 全体を見せる・余韻 |
| 横に流れる（左） | pan left | 場所の広がり |
| 横に流れる（右） | pan right | 場所の広がり |
| 上から下へ | tilt down | キャラ紹介・状況確認 |
| 下から上へ | tilt up | 建物・空への移行 |
| 固定 | static shot, locked off camera | 安定した場面 |
| 手持ちカメラ感 | slight handheld movement | リアリティ |

カメラムーブなしのほぼ静止に近い動画も使いどころはあります。人物がしゃべっているシーンや静かな場面では、ほぼ静止で口や目だけ微妙に動く動画が自然に見えます。

---

## キャラクターの動作指示

人物が動くシーンは、どんな動作をするかを書きます。

```
作業シーン：
「田中悠斗がコンテナを手際よく仕分けしている。
腕を動かしながら作業。リズムよい動き。」

語りかけシーン：
「田中悠斗が正面を向いてカメラに語りかけている。
口が動く。表情が変わる。自然な体の微動。」

感情シーン：
「幸子が驚いて口を開ける。体がわずかにのけぞる。
目が見開く。感情が体全体に出ている。」
```

「自然な動き」「リアルな動作」という抽象的な言葉より、具体的な動作を書いた方が意図した動きに近くなります。

---

## 5秒と3秒の使い分け

Seedanceは生成する動画の長さを指定できます。主に3秒・5秒のどちらかを選ぶことが多いです。

**5秒使うカット：**
- シーンの最初に来る場所説明のカット（工場外観、室内全体など）
- 感情的な山場（泣く・喜ぶ・怒る）
- キャラクターが大きく動くシーン（歩く・走る・作業する）
- 動画の最後のCTAカット

**3秒で十分なカット：**
- テンポよく情報を並べる転職系のシーン
- クローズアップの積み重ね（手・顔・物のアップ）
- 静止画に近い状態でいいシーン（書類を見ている・考えている）

悩んだら5秒を選ぶ方が、編集で短くカットできるので後から調整しやすいです。

---

## 生成が失敗したときの対処

Seedanceも完全ではなくて、意図した動きが出ないことはあります。

**顔が崩れる・人物が変形する：**
これはアニメ画像との相性の問題で、完全には防げません。複数回生成して一番良いものを選ぶ。

**ほとんど動かない：**
動きの指示が弱い。カメラムーブを追加するか、動作を具体的に書き直す。「slow zoom in」「slight camera movement」を追加するだけでも変わります。

**意図しない動きになる：**
プロンプトに余分な解釈の余地が多い。短くシンプルにして、動いてほしい要素だけを書く。

**生成が途中で止まる・エラーになる：**
CapCutのサーバー負荷の問題。時間を置いてリトライ。

---

## コストを抑えながら品質を維持する戦略

CapCut内でのSeedance使用は45クレジット/5秒（約45円）です。

転職ショート（16カット×3〜5秒）を全カット動画化すると：
- 最短3秒で計算：16×27円 ≈ 430円
- 最長5秒で計算：16×45円 ≈ 720円

幸子ロング（120カット）だと：
- 最短3秒：120×27円 ≈ 3,240円
- 最長5秒：120×45円 ≈ 5,400円

コストを抑える方法として「全カットを動画化しない」というアプローチがあります。

**動画化が効果的なカット：**
- 人物が動くシーン（歩く・作業する・感情表現）
- 冒頭・ラストのインパクトが必要なカット
- 視聴者の印象に残したい山場のシーン

**ケンバーンズ（Ken Burns）で十分なカット：**
- 背景だけのカット（工場外観・空・風景）
- テキスト補足が主役のカット
- 短いつなぎのカット

背景・風景カットはケンバーンズ（ゆっくりズームイン/アウト）でも十分な場合が多く、コストをかなり抑えられます。

---

## まとめ

1. **Seedanceのプロンプトは「動き」を書く**
   静止画の説明ではなく、映像として何が動くかを書く

2. **カメラムーブを指定する**
   push in / pull out / pan / tilt / static など

3. **5秒と3秒を使い分ける**
   山場・場所説明は5秒、テンポ重視は3秒

4. **全カットを動画化しなくていい**
   人物シーンを優先して、背景はケンバーンズで節約

5. **生成失敗は複数回試して選ぶ**
   完璧を求めて無限リトライしない

## 【補足】外部リサーチで見つけたSeedance 2.0の追加テクニック

ここからは自分の経験だけでなく、外部のリサーチで見つけた情報も含めて書きます。

### プロンプトの最適長は35〜80ワード

これは公式ガイドにも書いてある情報ですが、プロンプトは長すぎても短すぎてもダメで、35〜80ワード（英語）が最も安定する範囲です。

日本語の場合は50〜120文字が目安になります。

### 推奨されているプロンプト構造

```
[ショットタイプ] + [被写体・人物] + [アクション] + [環境] + 
[カメラムーブ] + [照明] + [ビジュアルスタイル]
```

実際の例：
```
Medium shot. Young Japanese man in white work uniform, 
sorting packaged bread into containers, moving efficiently. 
Bright factory floor, red and blue plastic containers visible. 
Slow push in. Fluorescent lighting. Clean anime illustration style.
```

### カメラ指示は1つ＋1つに絞る

複数のカメラ動作を同時に指定すると、モデルが混乱します。

```
NG：「slow zoom in + pan right + tilt up」（多すぎる）
OK：「slow zoom in, slight pan right」（主＋副の2つまで）
```

「slow, smooth, stable, gradual, gentle」というリズミカルな形容詞が効きやすいです。

### 顔崩れが起きるときの対処

キャラクターの顔が途中で変わってしまう（アイデンティティドリフト）問題に対して、公式が推奨しているのは：

1. モーション値を「2」に下げる
2. キャラクターのリファレンス画像（CREF）をアップロードして参照させる

これで顔の安定性がかなり改善するケースが多いです。

### モーションリファレンス機能

実はSeedanceには「動きのある動画クリップをアップロードして、そのモーションだけキャラクターに適用する」という機能があります。

例えば「歩いている人物の動画クリップ」をアップロードして「この動きをこのアニメキャラクターに適用して」という指示ができる。モーション精度は約90%。

複雑な動きを出したいときに活用できます。

### 2000本以上の無料プロンプト集

GitHubに「awesome-seedance-2-prompts」というリポジトリが公開されていて、アニメ・広告・映画風など様々なジャンルのSeedanceプロンプトが2000本以上無料で使えます。

参考になるプロンプトが大量にあるので、一度見ておくと引き出しが増えます。
